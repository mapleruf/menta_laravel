開発環境セットアップ

開発環境セットアップ

それでは、実際にLaravelでアプリケーションを作成していきましょう。

動作のための環境設定

Laravelは、RDBやRedisなどのデータベース連携やセッション／クッキー、任意でAWSなどのサービスとの連携も行うことができます。 まずは、データベース操作を行うことができるように、データベース設定を行いましょう。

環境設定は、下記のファイルに追記／上書きを行います。

WebApp/.env

先頭にドットがついているファイルは隠しファイルとして扱われるため、Windowsのフォルダ上では表示されないかもしれません。 その場合は、隠しファイルを表示できるようにしてください。
AtomやVisualStudioCodeなどのエディタを使用している場合は、リストとして表示可能ですので、そのまま選択して編集を行いましょう。

.env ファイルを開くと下記のような記述を見つけることができると思います。これがデータベースの設定箇所になります。

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=webapp
DB_USERNAME=root
DB_PASSWORD=

接続先やユーザー名、パスワードは、ご自身の環境にあわせて内容を変更してください。

開発のための簡易サーバー起動

Laravelには動作確認のための簡易サーバー機能が付属しています。
プロジェクトフォルダ内で下記のコマンドを実行することで、ローカルPC内に簡易的なサーバーを用意することができます。

php artisan serve

コマンドを実行すると下記のようなコメントが表示されます。

Laravel development server started: http://127.0.0.1:8000

これはhttp://127.0.0.1:8000もしくはhttp://localhost:8000というURLでLaravelのプログラムにアクセス可能にしましたよという意味です。 動作を確認する場合は、このコマンドで簡易サーバーを起動してブラウザからアクセスするようにしてください。

z また、ターミナルやコマンドプロンプトからphp artisan serve を起動するとログ表示モードに切り替わり待機モードになります。 これはアクセスされたログを随時表示してくれるモードですが、ターミナルやコマンドプロンプトを終了した場合は簡易サーバーも終了してしまいますので注意してください。

意図的に簡易サーバーを終了させたい場合、Ctrl + C で終了させることができます。

Laravelの仕組み

Laravelでは、「ルーティング」「コントローラー」「モデル」「ビュー」が1セットとなって、 データベースを利用した1つの画面を作成および表示することができるようになっています。
まず、ルーティングでHTTPリクエスト（GET、POSTなど）に対応するコントローラと、そのメソッドを定義します。 ユーザーからのアクセスがあった場合、そのルーティングに対応したコントローラのメソッドを呼び出すわけです。

コントローラーからデータベースを参照したい場合、モデルを利用してデータ取得参照します。また、登録／更新などもここで行われます。
モデルで取得したデータはコントローラに受け渡しされ、そのデータを元に表示のためのデータ加工を行います。 加工が行われたデータはコントローラーから指定のビューに渡され、画面表示がされます。

また、コントローラからビューを介さずユーザーにデータを開示する事もできます。 これは通常、API（アプリケーションプロトコルインターフェイス）と呼ばれます。返却するデータはJSONやXMLと呼ばれる形式で開示され、利用される際はAjaxなどを利用してデータを受け取ることになります。
APIについての詳しい内容についてはホチほど作りながら解説していきますので、今は「そんなのものがあるんだな」程度に覚えておいてください。
