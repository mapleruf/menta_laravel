サービスプロバイダ

サービスプロバイダ

https://readouble.com/laravel/5.8/ja/providers.html

サービスプロバイダとは、Laravel全体の起動処理における初めの心臓部です。 Laravelのコアサービスはすべてサービスプロバイダを利用して初期起動処理を行っています。
これは、サービスコンテナの結合やイベントリスナ、フィルター、ルートなどを登録することを指します。 これだけではよくわからないと感じるかも知れませんが、要するにアプリケーションを処理を決定する上での 中心部分のようなものとイメージして下さい。

サービスプロパイダの登録は config/app.php ファイルに記述してあります。まずはファイルを開いて providersという配列の要素確認をしてみましょう。 ここに記述されているクラス（名前空間）がサービスプロバイダです。そしてこれはアプリケーションの起動時に読み込まれます。

    'providers' => [

        /*
         * Laravel Framework Service Providers...
         */
        Illuminate\Auth\AuthServiceProvider::class,
        Illuminate\Broadcasting\BroadcastServiceProvider::class,
        Illuminate\Bus\BusServiceProvider::class,
        Illuminate\Cache\CacheServiceProvider::class,
        Illuminate\Foundation\Providers\ConsoleSupportServiceProvider::class,
        Illuminate\Cookie\CookieServiceProvider::class,
        Illuminate\Database\DatabaseServiceProvider::class,
        Illuminate\Encryption\EncryptionServiceProvider::class,
        Illuminate\Filesystem\FilesystemServiceProvider::class,
        Illuminate\Foundation\Providers\FoundationServiceProvider::class,
        Illuminate\Hashing\HashServiceProvider::class,
        Illuminate\Mail\MailServiceProvider::class,
        Illuminate\Notifications\NotificationServiceProvider::class,
        Illuminate\Pagination\PaginationServiceProvider::class,
        Illuminate\Pipeline\PipelineServiceProvider::class,
        Illuminate\Queue\QueueServiceProvider::class,
        Illuminate\Redis\RedisServiceProvider::class,
        Illuminate\Auth\Passwords\PasswordResetServiceProvider::class,
        Illuminate\Session\SessionServiceProvider::class,
        Illuminate\Translation\TranslationServiceProvider::class,
        Illuminate\Validation\ValidationServiceProvider::class,
        Illuminate\View\ViewServiceProvider::class,

        /*
         * Package Service Providers...
         */

        /*
         * Application Service Providers...
         */
        App\Providers\AppServiceProvider::class,
        App\Providers\AuthServiceProvider::class,
        // App\Providers\BroadcastServiceProvider::class,
        App\Providers\EventServiceProvider::class,
        App\Providers\RouteServiceProvider::class,

    ],

ほとんどのプロバイダは必ず読み込まれるとは限りません。 そのプロバイダが提供するサービスが実際に必要なときにのみ読み込まれる「遅延」プロバイダと呼ばれるものとなっています。

サービスプロバイダの記述方法

サービスプロバイダはIlluminate\Support\ServiceProviderクラスを拡張（継承）していて、 ほとんどのサービスプロバイダでregisterとbootメソッドを使うことができます。

まずは、下記のコマンドを使ってサービスプロバイダを作成してみましょう。 作成ができたらapp/ProvidersフォルダにTaskServiceProvider.phpが作成されていることを確認してください。

php artisan make:provider TaskServiceProvider

早速、内容を確認してみます。
確かに registerメソッドとbootメソッドが定義されていますね。

<?php

namespace App\Providers;

use Illuminate\Support\ServiceProvider;

class TaskServiceProvider extends ServiceProvider
{
    /**
     * Register services.
     *
     * @return void
     */
    public function register()
    {
        //
    }

    /**
     * Bootstrap services.
     *
     * @return void
     */
    public function boot()
    {
        //
    }
}

Registerメソッド

registerメソッドにはサービスコンテナに何かを結合する処理だけを記述します。
それ以外の処理を記述してしまうと、サービスプロバイダがまだロードしていないサービスを意図せず使ってしまったり 意図した動きにならないかも知れません。

基本的なサービスプロバイダを見てみましょう。
サービスプロバイダメソッド中であれば、ServiceProviderクラスから継承されるappプロパティを利用して サービスコンテナへアクセスすることができます。
下記のではサービスコンテナとしてTask\Connectionの実装を定義している想定です。

<?php

namespace App\Providers;

use Task\Connection;
use Illuminate\Support\ServiceProvider;

class TaskServiceProvider extends ServiceProvider
{
    /**
     * Register services.
     *
     * @return void
     */
    public function register()
    {
        $this->app->singleton(Connection::class, function ($app) {
            return new Connection(config('task'));
        });
    }

Bootメソッド

ビューコンポーザをサービスプロバイダで登録する必要がある場合はbootメソッドを利用します。
このメソッドは、すべてのサービスプロバイダが登録し終えてから呼び出されます。 つまりフレームワークにより登録された他のサービス全てにアクセスできるということです。

ビューコンポーザはビューが表示される時に呼び出されるコールバックメソッドもしくはクラスメソッドのことです。 これは、ビューが表示されるたびに結合したい情報がある場合はビューコンポーザを使ってロジックを一箇所にまとめると便利です。

    /**
     * Bootstrap services.
     *
     * @return void
     */
    public function boot()
    {
        view()->composer('view', function () {
            //
        });
    }

bootメソッドでの依存注入

サービスプロバイダのbootメソッドでは、依存をタイプヒントで指定することができます。 このときサービスコンテナが必要な依存を自動的に注入してくれます。

public function boot(ResponseFactory $response)
{
    $response->macro('caps', function ($value) {
        //
    });
}
