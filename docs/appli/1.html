サービスコンテナ

サービスコンテナ

https://readouble.com/laravel/5.8/ja/container.html

サービスコンテナは、クラス間の依存を管理するためのツールです。
依存注入というのは「コンストラクターかセッターメソッドを使って、外部から、特定のクラスに依存しているクラスを注入する」 という意味で使われています。
要するに、そのクラスに必要なクラスを持ってくるための機能だと覚えておくと良いでしょう。

簡単な例を上げてみたいと思います。

<?php

namespace App\Http\Controllers;

use App\User;
use App\Repositories\UserRepository;
use App\Http\Controllers\Controller;

class UserController extends Controller
{
    protected $users;

    public function __construct(UserRepository $users)
    {
        $this->users = $users;
    }

    public function show($id)
    {
        $user = $this->users->find($id);

        return view('user.profile', ['user' => $user]);
    }
}

この例で、UserControllerはデータベースからユーザー情報を取得しています。 そこで、ユーザーを取得できる機能を持ったサービスを外部から注入しています。
データベースからユーザー情報を取得する場合、これまでであればモデルクラスをコントローラ内で使用していましたが ここではUserRepositoryというクラスが外部から注入されているので、それを経由した他の処理へ簡単に書き換えることができます。

サービスコンテナの説明についてはここまでですが、次のページに記載する「サービスプロパイダ」ととても深い関係を持っています。 あまり深追いをせず、次のページに進んでサービスプロバイダを学習しましょう。
