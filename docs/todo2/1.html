認証機能

認証機能

https://readouble.com/laravel/5.8/ja/authentication.html

Laravelでは簡単に認証をとても実装することができます。 認証のための設定ファイルは config/auth.php に用意されていて、認証処理のためのオプションも用意されています。

Laravelの認証機能はガードとプロバイダを中心に構成されています。

ガードは各リクエスト毎にどのようにユーザーを認証するかを定義します。 例えば、セッションストレージとクッキーを使いながら状態を維持するSessionガードがそれにあたります。

一方でプロバイダは永続ストレージからどうやってにユーザーを取得するかを定義します。 Laravelでは、Eloquentとクリエビルダを使用してユーザー情報を取得する事ができますが 必要に応じて自由にプロバイダを追加することもできるようになっています。

ルートの定義

ここまでの学習の中で、ルーティング（ルート）を定義するには web.php を編集してきましたが、 下記のコマンドをターミナルまたはコマンドプロンプトから実行することで認証に関するルートの定義を 簡単に行うことができます。

php artisan make:auth

Laravel 6.x 以降のバージョンではこの方法が利用できなくなっていますので、もしあえてLaravel 6.x 以上のバージョンで 学習を進められている場合は、下記のサイトを参考に定義すると良いでしょう。 Laravel 6.x 認証
https://readouble.com/laravel/6.x/ja/authentication.html
Laravel 7.x 認証
https://readouble.com/laravel/7.x/ja/authentication.html

このコマンドを実行すると、下記のフォルダを作成し必要なファイルが自動で作成されます。 コマンドを実行して、エディタなどからフォルダとファイルが作成していることを確認してみましょう。

app/resources/views/auth/
app/resources/views/layouts/
app/Http/Controllers/Auth

また、下記のコマンドを実行して定義済みのルーティングを確認してみてください。

php artisan route:list

このコマンドは、定義済みのルートを一覧として表示することができます。 この中に下記のAuthに関するルートが定義されていることを確認しましょう。

次に認証画面の確認を行いたいのですが、これまでの学習の中で設定したルート設定が邪魔をして表示ができません。
試しにhttp://localhost:8000/loginにアクセスするとエラーが表示されると思います。

これは、ルート設定の{id}にloginという文字が値としてアサインされてルーティングされてしまうため 詳細ページ処理が実行されてしまうからです。これを回避するため、ルート設定を下記のようにtask専用に変更してしまいましょう。

// タスク一覧画面
Route::get('task/', 'TodoController@index')->name('task.index');
// タスク新規作成画面
Route::get('task/create', 'TodoController@create')->name('task.create');
// タスク編集画面
Route::get('task/edit/{id}', 'TodoController@edit')->name('task.edit');
// タスク詳細画面
Route::get('task/{id}', 'TodoController@show')->name('task.detail');
// タスク登録処理
Route::post('task/store', 'TodoController@store')->name('task.store');
// タスク更新処理
Route::post('task/update/{id}', 'TodoController@update')->name('task.update');
// タスク削除処理
Route::post('task/delete', 'TodoController@delete')->name('task.delete');
Auth::routes();

Route::get('/home', 'HomeController@index')->name('home');

ここで見慣れない行が２行追加されている事に気づいたかと思います。これが認証に関するルートの設定を定義しているものになります。
この位置だと見づらいので、下記のように一番上に書き直しておきましょう。

Auth::routes();

Route::get('/home', 'HomeController@index')->name('home');

// タスク一覧画面
Route::get('task/', 'TodoController@index')->name('task.index');

Route::get('task/create', 'TodoController@create')->name('task.create');
// タスク編集画面
Route::get('task/edit/{id}', 'TodoController@edit')->name('task.edit');
// タスク詳細画面
Route::get('task/{id}', 'TodoController@show')->name('task.detail');
// タスク登録処理
Route::post('task/store', 'TodoController@store')->name('task.store');
// タスク更新処理
Route::post('task/update/{id}', 'TodoController@update')->name('task.update');
