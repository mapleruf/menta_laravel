認証機能その２

認証機能その２

認証機能では、ログインに成功すると/homeのURLへリダイレクトするように標準で設定されています。
これをカスタマイズするには、app/Http/Controllers/Auth 配下にある LoginController.php、RegisterController.php、 ResetPasswordController.php、VerificationController.php それぞれの redirectTo プロパティの値をURIやURL+として設定し直すことで、 リダイレクト先のページを変更することで変更できます。

class LoginController extends Controller
{
    /*
    |--------------------------------------------------------------------------
    | Login Controller
    |--------------------------------------------------------------------------
    |
    | This controller handles authenticating users for the application and
    | redirecting them to your home screen. The controller uses a trait
    | to conveniently provide its functionality to your applications.
    |
    */

    use AuthenticatesUsers;

    /**
     * Where to redirect users after login.
     *
     * @var string
     */
    protected $redirectTo = '/home';

今回はユーザー登録とログインした後にタスク一覧に遷移させたいのでLoginController.php、RegisterController.phpを 下記のように変更してしまいましょう。

    protected $redirectTo = '/task';

ログインID項目の変更

今回は修正を行いませんが、Laravelはデフォルトでinputタグの name="email" をデータベースの email カラムと照合して 認証処理をするようになっています。これをカスタマイズしたい場合は、LoginControllerにusernameメソッドを追加します。

public function username()
{
    return 'username';
}

この return として任意の名前を文字列として指定することで、その名前をname属性として持つinputタグ、 データベースのカラム名を認証のユーザーIDをして照合するように変更ができます。

inputタグのname属性とデータベースのカラム名は同じ名前のものが利用されます。

ユーザー登録に関するバリデーション等のカスタマイズ

新しいユーザーを登録する場合に入力してもらうinputタグのname属性を変更したり、 データベースに新しいユーザーレコードを登録する方法をカスタマイズしたい場合は、 RegisterControllerを変更します。

RegisterController の validatorメソッド には新規ユーザーに対するバリデーションルールが定義してあります。

    /**
     * Get a validator for an incoming registration request.
     *
     * @param  array  $data
     * @return \Illuminate\Contracts\Validation\Validator
     */
    protected function validator(array $data)
    {
        return Validator::make($data, [
            'name' => ['required', 'string', 'max:255'],
            'email' => ['required', 'string', 'email', 'max:255', 'unique:users'],
            'password' => ['required', 'string', 'min:8', 'confirmed'],
        ]);
    }

また、RegisterController の createメソッド は新しいApp\Userモデルのレコードをデータベースに登録するために利用されます。 必要に応じて自由に内容を変更することができます。

    /**
     * Create a new user instance after a valid registration.
     *
     * @param  array  $data
     * @return \App\User
     */
    protected function create(array $data)
    {
        return User::create([
            'name' => $data['name'],  // <- inputタグのname="name" の値をDBのnameカラムに登録している
            'email' => $data['email'],
            'password' => Hash::make($data['password']),
        ]);
    }

認証済みユーザーの取得

ログイン済みのユーザー情報を取得したい場合は、Authファサードを使うことができます。

use Illuminate\Support\Facades\Auth;

// 現在認証されているユーザーの取得
$user = Auth::user();
// 現在認証されているユーザーのID取得
$id = Auth::id();

また、ユーザーが認証済みであればIlluminate\Http\Request経由で、ユーザーへアクセスすることもできます。
Illuminate\Http\Request はコントローラーを作成した時点で既に use されているため、メソッドの引数として そのまま利用することができます。

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class TodoController extends Controller
{
    /**
     * ユーザープロフィールの更新
     *
     * @param  Request  $request
     * @return Response
     */
    public function index(Request $request)
    {
        // $request->user()は認証済みユーザーのインスタンスを返す
        $user_info = $request->user();
        echo $user_info->id;
    }
}
