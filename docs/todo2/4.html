テーブルのリレーション

テーブルのリレーション

https://readouble.com/laravel/5.8/ja/eloquent-relationships.html

データベースはほとんど場合、テーブル間の関連（リレーション）を持たせることができます。
これは例えば、ユーザーは複数のタスクを作成し、そのタスクは作成したユーザーのものであるといったものです。 Eloquentを利用すると、そういったリレーションを簡単に表現することができます。

リレーションの使い方

リレーションを実現するには、どのテーブル（モデル）とのリレーションを持っているという情報をモデルクラスのメソッドとして定義します。 そして、その参照には下記のような方法が存在します。
現時点では、「こういう物があるんだ」ということだけ、覚えておいてください。

// １ユーザーと関連するタスクを１件取得
$user = new User;
$user->first()->task;

// ユーザーと関連するタスクのうち id カラムの値が 2 以上の情報を取得
$user = new User;
$user->task()->where('id', '>=', 2)->get();

▶逆関係の定義

上記では、Userモデルに対するTaskモデルの１対１のリレーションについての定義を書きました。

User -> Task
※ １人のユーザーは１つのタスクを持っている

これは逆の位置関係として定義することもできます。

Task <- User
※ １つのタスクは１人のユーザーに保持されている

hasOneの逆のリレーションを定義するには、belongsToメソッドを使います。
ここで注意点として、先程はUesrモデルにメソッドを追加しましたが今回は逆からの位置関係を定義するため、 Taskモデルにメソッドを追加する必要があります。

<?php

namespace App;

use Illuminate\Database\Eloquent\Model;

class Task extends Model
{

    public function user()
    {
        return $this->belongsTo('App\User');
    }

多対多

多対多の関係はhasOneやhasManyによるリレーションよりも多少複雑な関係になります。 関係の例としては、ユーザーは複数の権限を持っていて、権限もまた大勢のユーザー紐付いているされるというイメージになります。
例えば、"管理者"としてのの権限を持つことの出来る１人とは限りません。

これは、users、roles、role_userという３テーブルでそれぞれの関係性をLaravelに認識させることができます。
roleuserテーブルはusersテーブルとrolesテーブルを結ぶ中間テーブルの役割を持っているのですが、名前の付け方に注意です。 これは、usersテーブルを担うUserモデル、rolesテーブルを担うRoleモデルをアルファベット順に並べて``で区切った名前にしなければいけません。
また、このテーブルはusersテーブルとrolesテーブルの紐付けのために使われるので、必ずuser_idカラムとrole_idカラムを保つ必要があります。

課題

rolesテーブルとrole_userテーブルを作成するマイグレーションファイルを作ってテーブルを追加してください。
rolesテーブルでは、権限IDを保存するidカラム（数値：AI）と権限名も保存するname（文字列）をカラムとして持つようにしてください。
また、role_userテーブルには作成者のユーザーIDを保存するための creator_id というカラムも追加しましょう。
テーブルの追加ができたら、rolesテーブルには「管理者」「担当者」の２つの権限を登録しておいてください。
テーブルの作成とレコードの作成ができたら、最後にRoleモデルを作成しておきましょう。


課題２

role_userテーブルにダミーデータを登録するシーダーを作成してください。
このとき、user_idカラムには1を固定、role_idカラムには1と2、creator_idカラムには1を固定で
2件のレコードを登録するようにしてください。
