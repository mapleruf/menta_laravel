少し進んだBladeの書き方

少し進んだBladeの書き方

https://readouble.com/laravel/5.8/ja/blade.html

@yieldディレクティブ - セクション読み込み -

@yieldは前回のページで書いたように、セクションを呼び出すために利用します。

    <main style="margin-top: 6rem">
        <div class="container">
            @yield('content')
        </div>
    </main>

@yieldで表示するセクションが存在しない場合

@yieldを記述したものの指定したセクションが特定できない（定義されていない）場合もあると思います。 この場合は、第２引数を利用して別のViewを呼び出すことができます。

    <main style="margin-top: 6rem">
        <div class="container">
            @yield('content', View::make('welcome'))
        </div>
    </main>

View::make() は、指定されたviewファイルをresources/viewsフォルダ配下から参照するための方法で、 使い方としてはコントローラから return する時に利用する view メソッドを同じです。

コンポーネントとスロット - 外部Bladeの組み込み -

コンポーネントとスロットは、セクションとレイアウトと同じような機能ですが、コンポーネントとスロットのほうがより簡単に やりたいことを実現できるかも知れません。
このコンポーネントというのは、画面上に表示する部品のことを指しています。例えば、エラーを表示する<div></div>などですね。

では、実際にタスク作成画面にエラーコンポーネントを実装してみたいと思います。 resources/view/layouts フォルダ配下に error.blade.php を作成して下記の内容を記述してみてください。

{{-- 追加：エラー表示 --}}
@if ($errors->any())
<div class="alert alert-danger">
    {{ $slot }}
</div>
@endif

次に、create.blade.php にコンポーネント利用のための処理を追加します。

@extends('layouts.default')
@section('content')
    {{-- 追加：エラー表示 --}}
    @component('layouts.error')
    @foreach ($errors->all() as $error)
        <span>{{ $error }}</span><br>
    @endforeach
    @endcomponent
    <form action="{{route('task.store')}}" method="POST">
        @csrf
        タスク名：<input type="text" name="name" value="" placeholder=""><br>
        期日　　：<input type="date" name="deadline" value="" placeholder=""><br>
        詳細　　：<textarea name="detail"></textarea><br>
        <input type="submit" value="登録">
    </form>
@endsection

修正が終わったら、タスク追加画面から何も入力せずに登録ボタンを押してみましょう。 @component の部分にエラーが表示されたと思います。 このようにコンポーネントとスロットを利用することで、Bladeファイルとして記述した内容をそのページの一部として 表示／動作させることができるようになります。

コンポーネントに別名をつける

コンポーネントは@componentで利用すると書きましたが、任意の別名を付けて利用することもできます。
app/Providers/AppServiceProvider.php を開いて下記のように修正してみてください。

<?php

namespace App\Providers;

use Illuminate\Support\ServiceProvider;
use Illuminate\Support\Facades\Blade; // <- 追加

class AppServiceProvider extends ServiceProvider
{

...（省略）...

    /**
     * Bootstrap any application services.
     *
     * @return void
     */
    public function boot()
    {
        // 追加：layouts/alert.blade.php を @error をして利用できるようにする
        Blade::component('layouts.error', 'errors');
    }
}

このように、AppServiceProviderクラスのbootメソッドにコンポーネント互換の処理を書くことで 任意の名前で特定のコンポーネントの別名をつけることができます。
そして、create.blade.php を下記のように編集しましょう。

@extends('layouts.default')
@section('content')
    @errors
    @foreach ($errors->all() as $error)
        <span>{{ $error }}</span><br>
    @endforeach
    @enderrors
    <form action="{{route('task.store')}}" method="POST">
        @csrf
        タスク名：<input type="text" name="name" value="" placeholder=""><br>
        期日　　：<input type="date" name="deadline" value="" placeholder=""><br>
        詳細　　：<textarea name="detail"></textarea><br>
        <input type="submit" value="登録">
    </form>
@endsection
課題

タスク編集画面にもコンポーネントとスロットを使ったエラー処理を実装してみましょう。
実装ができたら、エラーが表示されることの確認も行ってください。

認証ディレクティブ - 認証情報の確認 -

現在のユーザーがログインしているのかどうかチェックするためのディレクティブも存在しています。

@auth
    // ユーザーは認証済み
@endauth

@guest
    // ユーザーは認証されていない
@endguest

これは、現在利用している layouts/default.blade.php のヘッダー部分荷を利用されていますね。 気になる場合は、確認してみましょう。

必要であれば、@authと@guestディレクティブ使用時に確認すべき認証ガードを指定することもできます。

@auth('admin')
    // ユーザーは認証済み
@endauth

@guest('admin')
    // ユーザーは認証されていない
@endguest

セクションディレクティブ - セクション存在判定 -

セクションがコンテンツを持っているかを判定したい場合は、@hasSectionディレクティブを使用します。

@hasSection('content')
    @yield('content')
@endif

Methodフィールド - フォームメソッドの補助的な指定 -

基本的にHTMLのフォームからは、GETとPOSTの２種類しか送信することができません。 これはGET以外のデータは、内部データとしてはPOSTデータとして送信されているからでもあります。
しかし、ルーティングのページでも学習したように、LaravelではPUT、PATCH、DELETE といったメソッドも サポートしています。これを意図的に判断させるために見かけ上のHTTP動詞を指定するための _methodフィールドというものを追加する必要があります。
@methodディレクティブでこのフィールドを生成できます。

<form action="{{route('task.store')}}" method="POST">
    @method('PUT')
    ...（省略）...
</form>

サブビューの読み込み - 外部Bladeの取り込み -

ビューから他のBladeビューを取り込むには @include が利用できます。ファイルの中をインポートする形で取り込むので 読み込んだビュー内では、取り込み元のビューで利用できる変数を使うことができます。

<div>
    {{-- resources/views/layouts/error.blade.php を取り込み --}}    
    @include('layouts.error')
</div>

include したビューが存在しない場合エラーとなりますが、これを回避するために@includeIfを使うことができます。

<div>
    {{-- resources/views/layouts/error.blade.php が存在していたら取り込み --}}    
    @includeIf('layouts.error')
</div>

もし条件に応じて、取り込むかどうかを決めたい場合は@includeWhenによって条件を指定して取り込むこともできます。

<div>
    {{-- resources/views/layouts/error.blade.php が存在していたら取り込み --}}    
    @includeWhen($errors->any(), 'layouts.error')
</div>

サブビューについては、「コンポートとスロット」との使い分けで混乱するかも知れません。 コンポートとスロットを使う場合は、「枠だけ決めて」内容はページ側で決定する。 サブビューを使う場合は、「すべての雛形を固定して定義する」と覚えて置くと良いと思います。

サブビューに別名をつける

サブビューにも、コンポーネントと同じく別名をつけることができます。 定義のやり方も、殆ど変わりありません。

app/Providers/AppServiceProvider.php を開いて下記のようにすることで、ビューでは任意の名前で利用ができます。

<?php

namespace App\Providers;

use Illuminate\Support\ServiceProvider;
use Illuminate\Support\Facades\Blade; // <- 追加

class AppServiceProvider extends ServiceProvider
{

...（省略）...

    /**
     * Bootstrap any application services.
     *
     * @return void
     */
    public function boot()
    {
        // 追加：layouts/alert.blade.php を @error をして利用できるようにする
        Blade::include('layouts.error', 'errors');
    }
}

使用例はこんな感じです。 コンポーネントと違って内容はすべて取り込みもとに書いてあるので、@enderrors などとする必要もありません。

@extends('layouts.default')
@section('content')
    @errors
    <form action="{{route('task.store')}}" method="POST">
        @csrf
        タスク名：<input type="text" name="name" value="" placeholder=""><br>
        期日　　：<input type="date" name="deadline" value="" placeholder=""><br>
        詳細　　：<textarea name="detail"></textarea><br>
        <input type="submit" value="登録">
    </form>
@endsection

レンダリングビュー - ビューの繰り返し -

@eachディレクティブを使うことで、ループとビューの読み込みを組み合わせることができます。 ただし、レンダリングされるビューでは変数が引き継がれないので、独自に変数を利用することになります。
一見不便そうに感じますが、これはリスト形式など特定のテンプレートで表示を作るのに最適です。

{{-- 特定要素のループ --}}
@each('view.name', $jobs, 'job')
{{-- ループする要素がなかったときのビューを指定 --}}
@each('view.name', $jobs, 'job', 'view.empty')
引数 意味
第１引数 ビュー名
第２引数 ループのための値
第３引数 ループで利用される変数名
第４引数 ループできない場合のビュー名

使用例として、実際にタスク一覧を@eachディレクティブで書き換えていってみましょう。
まず一覧表示用にresources/views/task/list.blade.php という名前のビューを作成します。

<tr>
    <td><a href="{{ route('task.detail', ['id' => $task->id]) }}">
        {{ $task->name }}
    </a></td>
    <td>{{ $task->detail }}</td>
    <td>{{ $task->deadline }}</td>
    <td><a href="{{ route('task.edit', ['id' => $task->id]) }}">編集</a></td>
    {{-- リンクを追加 --}}
    <td>
        <form method="post" action="{{ route('task.delete') }}">
            @csrf
            <input type="hidden" name="id" value="{{ $task->id }}">
            <input type="submit" name="" value="削除">
        </form>
    </td>
</tr>

次に、resources/views/index.blade.php を下記のように修正してしまいましょう。

@extends('layouts.default')
@section('content')
    <div>
        <a href="{{ route('task.create') }}">タスクの追加</a>
    </div>
    <table>
        <tr>
            <th>タスク名</th>
            <th>詳細</th>
            <th>期日</th>
        </tr>
        @each('task.list', $list, 'task')
    </table>
@endsection

物凄くシンプルになりましたね。 修正ができたらブラウザからタスク一覧画面にアクセスして表示を確認してみてください。

スタック

スタックとは、いうなれば退避させておくということです。 たとえば resources/view/layouts/defaoult.blade.php にはサイトレイアウトとしての雛形を記述していますね。 この中にスタックを使ってページ毎に呼び出すJavascriptを事前に退避させておきます。
これを必要に応じて、子ビューで呼び出すことができます。

defaoult.blade.php に下記のような記述を追加したとします。

@push('scripts')
    <script src="/example.js"></script>
@endpush

これを子ビューで利用する場合は、下記のようにすることができます。

<head>
    ...（省略）...
    @stack('scripts')
</head>

サービス注入 - サービスコンテナの利用 -

@injectディレクティブを使うことでサービスコンテナからサービスを呼び出してビューである買うことができます。 最初の引数はそのサービスを使うための変数名を任意で指定して、第２引数には利用したいサービスのクラス／インターフェイス名を指定します。

@inject('metrics', 'App\Services\MetricsService')
<div>
    Monthly Revenue: {{ $metrics->monthlyRevenue() }}.
</div>

ここで登場したサービスコンテナですが。「何のことやら・・？」となりますね。
一旦は「そんなのがあるんだ」とだけ覚えておきましょう。
