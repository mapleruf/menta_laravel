ルーティングの作成

ルーティングの作成

https://readouble.com/laravel/5.8/ja/routing.html

それでは、まずルーティングを作成してみましょう。
Webサイトのルーティングを行うには、下記のファイルを編集します。

routes/web.php

ファイルを開くと、下記のようなサンプルが既に記述してあるはずです。

Route::get('/', function () {
    return view('welcome');
});

これは、http://ドメイン名（またはIP）/というアクセスがされた場合、welcome.blade.php というビューファイルを表示しなさいということが書かれています。 この時の Route::get という書き方に注目してください。このgetというのは、HTTPリクエストのメソッドを表現しています。
通常のWebアクセスの場合はすべてGETリクエストとなるので、通常のアクセスの場合にwelcomeビューが呼び出されるということになります。
Route::xxxx にはいくつか種類が存在していて、対応させたいHTTPリクエストに応じてルーティングを使い分けを行うことが可能です。

例）一般的なルーティング

書き方 リクエストメソッド
Route::get GET
Route::post POST
Route::put PUT
Route::delete DELETE
Route::patch PATCH
Route::options OPTIONS
Route::resource GET/POST/PUT/DELETE/PATCH
※ 特殊ケース

沢山の例を上げましたが、Webアプリとしてのアクセスに限定すると通常はGETとPOSTくらいしか使わないでしょう。

ルーティングの書き方

ルーティングにはいろいろな記述方法がありますが、ここではベーシックな例をいくつか挙げたいと思います。

localhost へのアクセスを、TodoControllerのindexメソッドにルーティング

// http://localhost でアクセス出来るページ
Route::get('/', 'TodoController@index');

/detail/??? へのアクセスを、TodoControllerのshowメソッドにルーティング

// http://localhost/detail/123 などでアクセス出来るページ
Route::get('detail/{id}', 'TodoController@show');

この場合、???には任意の値を設定することを想定します。
???に設定された値はコントローラのshowメソッドで、引数 $id として取得することが可能になります。 メソッドの引数については何のことかまだ理解できないと思いますが、一旦は「そうなのか」と思っておいてください。

少し進んだルーティング方法

ルーティングには他にも書き方が存在しています。その一例を上げたいと思います。

【任意のパラメータ】
下記のようにした場合 ??? は任意のパラメータとすることができます。任意のパラメータというのはつまり、あってもなくても良いということです。

// http://localhost/detail/123 などまたは http://localhost/detail でアクセス出来るページ
Route::get('detail/{id?}', 'TodoController@show');

【正規表現制約】
引数idは、数値のみ指定可能

Route::get('detail/{id}', 'TodoController@show')
    ->where('id', '[0-9]+')

引数idは数値のみ、tagsは大文字小文字の英字のみ指定可能

Route::get('detail/{id}/{tags}', 'TodoController@show')
    ->where(['id' => '[0-9]+', 'tags' => '[a-z]+']);

【名前を付ける】
detail_show としてLaravel内で参照可能な名前（あだ名）をつける

Route::get('detail/{id}', 'TodoController@show')
    ->name('detail_show');

名前をつけたルーティングは、下記の様にコントローラで便利に利用することができます。

// URLの生成
$url = route('detail_show');

// detail_showへリダイレクト
return redirect()->route('detail_show');

または

// detail_showへリダイレクト
return redirect(route('detail_show'));

TODOアプリのルーティングを作成しよう

それでは実際に、TODOアプリのルーティングを作成してみましょう。
ここでは、一覧画面と詳細画面に着目して進めます。それ以外の機能が存在する場合は、それ以外の画面のルーティングも作成してみてください。

本来は、混乱を防ぐため各画面を作成するごとに定義したほうが良いですが 今回はイメージを持ってもらうために、一度に定義して進めていきます。

// Route::get('/', function () {
//     return view('welcome');
// });

// タスク一覧画面
Route::get('/', 'TodoController@index')->name('task.index');
// タスク新規作成画面
Route::get('/create', 'TodoController@create')->name('task.create');
// タスク編集画面
Route::get('/edit/{id}', 'TodoController@edit')->name('task.edit');
// タスク詳細画面
Route::get('/{id}', 'TodoController@show')->name('task.detail');
// タスク登録処理
Route::post('/store', 'TodoController@store')->name('task.store');
// タスク更新処理
Route::post('/update/{id}', 'TodoController@update')->name('task.update');
// タスク削除処理
Route::post('/delete', 'TodoController@delete')->name('task.delete');

おさらい

ルーティングは以下のように書く事ができる。
Route::HTTPメソッド名(‘ドメイン階層と引数’, ‘コントローラ名@メソッド名’)

‘コントローラ名@メソッド名’の部分には、処理を直接書くこともできます。
