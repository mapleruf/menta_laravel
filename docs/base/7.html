シーディング

シーディング

https://readouble.com/laravel/5.8/ja/seeding.html

次にモデルを利用したデータの取得を実装しますが、データが入っていないため何も取得することができません。 そのため、シーダーと呼ばれるダミーデータ生成機能を使ってダミーデータを作成します。

シーディング用ファイルの作成

シーダーは下記のようにして作成することができます。

php artisan make:seeder TasksTableSeeder

コマンドの実行に成功すると app/database/seeds/TasksTableSeeder.php というファイルが作成されるはずです。 早速、中身を確認します。

<?php

use Illuminate\Database\Seeder;

class TasksTableSeeder extends Seeder
{
    /**
     * Run the database seeds.
     *
     * @return void
     */
    public function run()
    {
        //
    }
}

ここに必要な情報を入力することでダミーデータや初期データを入力することが可能になります。

シーディングファイルの編集

先程作成したシーディングファイルを編集していきましょう。

今回はDBクラスも利用するためすでに存在している use の下に更に use しておきます。

use Illuminate\Database\Seeder;
use Illuminate\Support\Facades\DB;

また、今回はダミーデータを登録したいのでLaravelで利用できるFakerというクラスを利用します。

    public function run()
    {
        // Faker
        $faker = Faker\Factory::create('ja_JP');

        DB::table('tasks')->insert([
            'name' => $faker->lastName,
            'detail' => $faker->sentence,
            'deadline' => $faker->dateTimeThisMonth()
        ]);
    }

Fakerとは、上にも書いたとおりフェイクデータ（ダミーデータ）を作成するためのクラスです。 Face/Factory クラスの create メソッドに js_JP という引数を渡すことで極力日本語でのダミーデータを作成することができます。
また、tags カラムは json 型のカラムなため、json_encode をかけてjsonに変換しています。

Qiita / [Laravel5.1]Fakerチートシート https://qiita.com/tosite0345/items/1d47961947a6770053af

シーディングファイルの登録

シーディングファイルの作成／編集が終わるとそのファイルを実行の対象として登録する必要があります。 登録は、app/database/seeds/DatabaseSeeder.php を編集します。

初期では、下記のような内容になっていると思います。

<?php

use Illuminate\Database\Seeder;

class DatabaseSeeder extends Seeder
{
    /**
     * Seed the application's database.
     *
     * @return void
     */
    public function run()
    {
        // $this->call(UsersTableSeeder::class);
    }
}

この run というメソッドに下記のように書き加えることで TasksTableSeeder.php をシーディング対象として登録することができます。

    public function run()
    {
        // $this->call(UsersTableSeeder::class);
        $this->call(TasksTableSeeder::class);
    }

編集が終わったら下記のコマンドで 追加／変更されたクラスを認識できるようにしておきましょう。

composer dump-autoload

シーディングファイルの実行

それでは、先ほど作成したシーディングファイルを実行します。
下記のコマンドで実行してみてください。

php artisan db:seed

下記のような表示がされたら成功です。

Seeding: TasksTableSeeder
Database seeding completed successfully.

実行が完了したら、早速 tasks テーブルを確認しましょう。 新しく１レコードが追加されたのを確認できたと思います。 このように、シーダーを使うとダミーのデータを追加することができます。

また、この方法ではDatabaseSeederに登録されているシーダーはすべて実行されてしまうことになります。 もし単体のシーダーのみを実行したい場合は、下記のようにして単体のシーディングファイルを指定して実行できます。

php artisan db:seed --class=TasksTableSeeder
