テーブル操作

テーブル操作

Laravelを扱う上で、まず最初に躓くのはデータベースの操作部分です。 色々な書き方が存在しているため余計に混乱してしまうポイントだと思っています。

使い方や詳細などについては、使いながら覚えていくということにして簡単な説明のみをしておきたいと思います。

クエリビルダ

https://readouble.com/laravel/5.8/ja/queries.html

Laravelではテーブル操作の方法が２つ存在しており開発者はそのどちらか、もしくは両方を利用してデータベースにアクセスすることができます。 その中でも一番SQL文に近い形で書いていくことの出来る方法がクエリビルダになります。

【使用例】

$model = new Model;
// id カラムの値を最初の1件のみ取得
$result = $model->value('id');

$model = new Model;
// 100件ごとに処理をしながらデータを取得
$result = [];
$model->orderBy('id')->chunk(100, function($query) use (&$result) {
    foreach($query as $item) {
        $result[] = $item;
    }
});

Eloquent

https://readouble.com/laravel/5.8/ja/eloquent.html

クエリビルダともう一つの方法がEloquentを利用する方法です。 モデルの説明で「Eloquentという仕組みを利用してモデル操作を実現しています」と書きましたが、このEloquentにもデータベース操作のメソッドが存在しています。 ただし、いくつかの制約が存在するため、それを避けるためにはクエリビルダを使用していくという方法になるでしょう。

【使用例】

<?php

namespace App;

use Illuminate\Database\Eloquent\Model;

class Task extends Model
{
    public function get_task() {
        $ret = $this->get();
        return $ret;
    }
}
