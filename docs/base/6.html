モデル（Modelの作成）

モデル（Model）

https://readouble.com/laravel/5.8/ja/eloquent.html

ここからは少し複雑になります。 そのため、見出しの下に書いてあるURLをそれぞれ参照したり質問しながらテキストを進めるようにしてください。

コントローラーとはMVCモデルのMに該当します。 MVCの概念上、モデルという名前で呼びますがLaravelではEloquentという仕組みを利用してモデル操作を実現しています。

モデルは、簡単に言えばSQLとの連携を行うためのプログラムを扱う層になり、 データベースとのデータのやり取りはすべてモデルを通して行われます。

モデルの作成

まずは、モデルを作成して見ましょう。 Laravel プロジェクトフォルダの直下で下記のコマンドをターミナル（もしくはコマンドプロンプト）で実行して、 最初のモデルを作成してください。

php artisan make:model Task

コマンドの実行が成功すると、app/Http/Task.php というファイルが作成されるはずです。
Task.php の中を見ると下記のように Taskクラスで定義されているのが確認できると思います。

<?php

namespace App;

use Illuminate\Database\Eloquent\Model;

class Task extends Model
{
    //
}

Laravelでは、モデル名の小文字に複数形の「s」がついたものをデフォルトのテーブルとして関連性を持つたせるができます。 Taskモデルはとくにデータベース接続処理やテーブル名の指定をしなくてもtasksテーブルをそのまま扱うことが出来るということです。
また、モデルクラスをインスタンス化した時点で既にテーブルとの導線ができている状態を作ることが出来るため、 モデル = テーブルそのものと考えてもらっても問題はありません。

モデルメソッドの作成

https://readouble.com/laravel/5.8/ja/queries.html

先程作成したTaskクラスにメソッドを追加してみましょう。 ここで作成したメソッドは、コントローラーから呼び出して利用することになります。

下記を参考に get_task というメソッドを作成してみてください。

<?php

namespace App;

use Illuminate\Database\Eloquent\Model;

class Task extends Model
{
    public function get_task() {
        $ret = $this->get();
        return $ret;
    }
}

Laravelのドキュメントでは、下記のようにテーブルを直接ソースコードに書く形で説明してあります。

$users = DB::table('users')->get();

しかし書き方に違いはあれど、$this->get() と同じことをしています。
DBというクラスは、データベースの操作を担っているクラスです。 このクラスのtableメソッドを呼び出す際にテーブル名を引数として指定することで、テーブルを指定した状態でを作ることが出来るのですが、 先にも書いたように、クラス名とテーブルで既に導線を作ってしまっていますのでモデルとテーブル間は $this として省略することが出来ます。
そのため、当テキストでは $this として進めていきます。
