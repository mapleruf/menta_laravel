テーブルデータの参照と表示

ようやくダミーデータの登録までが完了しました。 それでは早速データベースから値を取得して画面に表示させてみたいと思います。

コントローラーの編集

まずはコントローラから、モデルのメソッドを呼び出します。 TodoController.php に use として App\Task を追加しましょう。
これは、tasksテーブルを利用するために Task.php というモデルを参照しますという宣言になります。

use Illuminate\Http\Request;
use App\Task; // <- これを追加

次に index メソッドを下記のように書き換えてください。

    public function index ()
    {
        $task = new Task;
        $list = $task->get_task();

        return view('index', [
            'title' => 'サンプル',
               'list'  => $list
        ]);
    }

ここでは、$taskという変数にTaskクラスをインスタンス化して代入し、インスタンス化したTaskクラスの get_task メソッドを呼び出しています。 また、その結果は $list 変数に格納され、list という変数名でビューから呼び出しできるようにしています。

ビューの編集

コントローラの編集が終わったら、コントローラから渡された list という変数をビューで利用するように編集していきます。

<html>
    <body>
        <h1>{{ $title }}ページ</h1>
        これは最初のViewです。<br>
        <ul>
        @foreach ($list as $item)
            <li>{{ $item->detail }}</li>
        @endforeach
        </ul>
    </body>
</html>

ここまでの編集が完了したら、ブラウザからlocalhost:8000にアクセスしてみましょう。 データベースに登録されているdetailカラムがリストとして表示されるのを確認できたと思います。

Task モデルの get_task メソッドでは、最終的に $this->get() というメソッドを使ってデータベースから値を取得しました。 この $this->get() ではオブジェクトとしてレコードの情報を取得しています。
PHPでは、オブジェクトのメソッドやプロパティを参照する場合 ->（アロー演算子）を使って参照するため、 ビューでもアロー演算子によって comment に格納された値を参照／表示をしているというわけです。

感の良い人は気づいたかもしれません。
Laravelではモデルで取得されたレーコードの情報は、実際のテーブルのカラム名として格納されてきます。 そのため、利用もカラム名を指定することで参照することが可能となっています。

おさらい

先程のビューを編集して、コメントの前に ID を出力するようにしてください。
また、IDとコメントの間にはわかりやすくタブ文字を表示するようにしましょう。
