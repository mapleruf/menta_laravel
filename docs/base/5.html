マイグレーション

マイグレーション

https://readouble.com/laravel/5.8/ja/migrations.html

コントローラー（C）、ビュー（V）ときて、今度はモデルの内容に入っていくのですが、その前にやっておくべきことがあります。 それは、データベースの準備です。

直接データベースにテーブルを作成しても良いですが、Laravelにはマイグレーションというテーブルの操作履歴を管理するための仕組みがあります。 これは、テーブル作成、テーブル変更、テーブル削除など順序性を持ったでデータベースの管理にとても適しています。

マイグレーションファイルの作成

早速マイグレーションファイルを作成して見ましょう。

Laravel プロジェクトフォルダの直下で下記のコマンドをターミナル（もしくはコマンドプロンプト）で実行して、 最初のマイグレーションファイルを作成してください。

php artisan make:migration create_tasks_table

コマンドの実行が成功すると、app/database/migrations/xxxx_xx_xx_xxxxxx_create_tasks_table.php というファイルが作成されるはずです。 まずは中身を確認してみましょう。

<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

class CreateSampleTable extends Migration
{
    /**
     * Run the migrations.
     *
     * @return void
     */
    public function up()
    {
        Schema::create('tasks', function (Blueprint $table) {
            $table->bigIncrements('id');
            $table->timestamps();
        });
    }

    /**
     * Reverse the migrations.
     *
     * @return void
     */
    public function down()
    {
        Schema::dropIfExists('tasks');
    }
}

まず、注目するのはupメソッド、downメソッドかと思います。 これらのメソッドはそれぞれが下記のような場合に実行されます。

upメソッド
php artisan migrate が実行されたとき
downメソッド
php artisan migrate:rollback が実行されたとき

マイグレーションファイルの編集

上記の通り、作成したてのマイグレーションファイルには簡単なカラム追加の情報のみが格納されています。 次にすべきは、作成するテーブルに必要な情報を追加編集していくことです。

では、早速マイグレーションファイルのupメソッドを編集してみましょう。

    public function up()
    {
        Schema::create(tasks, function (Blueprint $table) {
            $table->increments('id');
            $table->string('name', 255);
            $table->timestamps();
        });
    }

ここでは、idカラムをBIG INTEGER型のオートインクリメント指定からINTEGER型のオートインクリメント指定に変更し、 nameというVARCHAR型(255)のカラムを追加しました。

この他にもいろいろな型指定が存在しています。詳細は、Laravel和訳ドキュメントの「カラム作成」を参考にしましょう。

https://readouble.com/laravel/5.8/ja/migrations.html

最後に下記にように編集しておきましょう。

    public function up()
    {
        Schema::create('tasks', function (Blueprint $table) {
            $table->increments('id');
            $table->string('name', 255);
            $table->string('detail', 255);
            $table->datetime('deadline', 0)->nullable();
            $table->timestamps();
        });
    }

マイグレーション実行

マイグレーションファイルの作成が完了したら、マイグレーションを実行して実際にテーブルを作成します。
実行は下記のコマンドで行うことができます。

php artisan migrate

また、下記の行がターミナルまたはコマンドプロンプト上に表示されれば実行は成功しています。

Migrated:  2014_10_12_000000_create_users_table (0.05 seconds)
Migrating: 2014_10_12_100000_create_password_resets_table
Migrated:  2014_10_12_100000_create_password_resets_table (0.02 seconds)

実行ができたら、データベースを確認してtasksテーブルが作成されていることを確認してください。

マイグレーション取り消し

マイグレーションとは、履歴を残しながらデータベースを管理することのできる仕組みです。 マイグレーションによって作成されたテーブルは、直接削除することもできますが、 作成の際と同じ様に magrate コマンドを使ってロールバックさせることで、テーブル作成の履歴をたどって、 整合性を保つことができます。

php artisan migrate:rollback

このとき、すでにテーブル内に作成されているデータは削除されてしまいますので 必要なデータはSQLダンプなどでバックアップを撮っておく必要があるのとに注意しましょう。

それでは早速先程マイグレーションした結果をロールバックを試してみてください。

Rolling back: 2020_01_31_051530_create_tasks_table
Rolled back:  2020_01_31_051530_create_tasks_table (0.08 seconds)
Rolling back: 2019_08_19_000000_create_failed_jobs_table
Rolled back:  2019_08_19_000000_create_failed_jobs_table (0 seconds)
Rolling back: 2014_10_12_100000_create_password_resets_table
Rolled back:  2014_10_12_100000_create_password_resets_table (0.01 seconds)
Rolling back: 2014_10_12_000000_create_users_table
Rolled back:  2014_10_12_000000_create_users_table (0.01 seconds)

上記のような結果が表示されたらロールバックは完了です。

課題

create_tasks_table.php に integer型の rep_id カラムを追加してマイグレーションを実行してください。
実行できたら、rep_id カラムを持ったテーブルが作成されていることを確認しましょう。

各民ができたら、ロールバックを実行して、追加した rep_id の行は削除しておきましょう。
その後、再度マイグレーションを実行してください。
