コントローラー（Controller）の作成

コントローラー（Controller）の作成

https://readouble.com/laravel/5.8/ja/controllers.html

コントローラーとはMVCモデルのCに該当します。
これは、司令塔などと表現されることもあり、ユーザーがまず最初にアクセスする場所です。 コントローラーを経由してモデルを呼び出し、ビューを呼び出されることで、最終的にページが表示されます。

コントローラーの作成

まずはコントローラを作成して見ましょう。
Laravel プロジェクトフォルダの直下で下記のコマンドをターミナル（もしくはコマンドプロンプト）で実行して、 最初のコントローラを作成してください。

php artisan make:controller TodoController

コマンドの実行が成功すると、app/Http/Controllersフォルダの配下にTodoController.phpというファイルが作成されるはずです。

クラスを利用するアプリでは殆どの場合、クラス名とファイル名は同じものを使用します。 そのためTodoController.php の中を見ると下記のようにTodoControllerクラスで定義されているはずです。

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class TodoController extends Controller
{

}

コントローラーメソッドの作成

先程作成したTodoControllerクラスにメソッドを追加してみましょう。
ルーティングの章で作成したアクション名と追加されるメソッド名が一致する場合は、 そのルーティングが適用され任意のURLでアクセスすることが可能になります> ルーティングが設定されていない場合は、内部処理のためのメソッドとしてのみ利用することが可能です。

下記を参考に index というメソッドを作成して見てください。

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class TodoController extends Controller
{
    public function index ()
    {
        return [123, 456, 789];
    }
}

作成できたら、早速 Laravelを起動しlocalhost:8000にアクセスしてみましょう。
下記のような文字がページに表示されたのでは無いでしょうか？

[123, 456, 789]

これはアクセス結果としてJSONというデータ形式で結果を表示しています。

Laravelでアクセス結果を表示する場合returnを利用するのですが、 ビュー表示用のメソッドの代わりに値や配列がreturnされた場合、それをJSONデータとして表示することができます。 これは一般的に、APIなどで利用される方法です。

ルーティングされたコントローラメソッドの作成

ルーティングを決めたアプリのコントローラも作成してみます。
ルーティングでは、TodoControllerのindexとshowという２つのメソッドに対するルーティングを作成しましたので、それらを実装していきます。

class TodoController extends Controller
{
    public function index ()
    {
        return 'indexメソッド';
    }

    public function show ()
    {
        return 'showメソッド';
    }
}

作成できたら、それぞれのメソッドにルーティングされたURLにアクセスして表示結果を確認しておきましょう。

index メソッドのルーティング

Route::get('/', 'TodoController@index');


show メソッドのルーティング

Route::get('/{id}', 'TodoController@show');

show メソッドへは引数が必要なことに注意してください。 http://localhost:8000/ではアクセスできませんが、http://localhost:8000/1などとすることでアクセスが可能になります。

おさらい

ルーティングの時の引数{id}は、URLの一部として指定する。
