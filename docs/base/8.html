ファクトリ

ファクトリ

https://readouble.com/laravel/5.8/ja/database-testing.html

Laravelにはデータベースのテストのために、ファクトリという機能が存在しています。 これは、一括で大量のデータをデータベースに登録したりする場合に便利です。

ファクトリファイルの作成

それでは早速、ファクトリの作成をしてみましょう。

php artisan make:factory TaskFactory

このコマンドの実行が完了すると app/database/factories/TaskFactory.php が生成されます。 初期として下記のような内容が書かれていますが、ここにシーディングで書いたような内容を書くことになります。

use App\Model;
use Faker\Generator as Faker;

$factory->define(Model::class, function (Faker $faker) {
    return [
        //
    ];
});

ファクトリファイルの編集

早速、シーディングファイルの内容を書き写していきましょう。ただし、Faker については既に定義されているためそれを利用します。 App\ModelもApp\Taskに書き換わりますので注意しましょう。

use App\Task;
use Faker\Generator as Faker;

$factory->define(Task::class, function (Faker $faker) {
    return [
        'name' => $faker->lastName,
        'detail' => $faker->sentence,
        'deadline' => $faker->dateTimeThisMonth()
    ];
});

ここで、DB::table や insert を含んでいないことに注意してください。 このファイルに書くべき内容はあくまでも、テーブル初期値の定義のみであり、テーブルそのものの情報ではありません。

シーディングファイルの編集とファクトリの利用

ファクトリの編集が終わったら、それをシーディングファイルで使用します。
ファクトリーそのものとしての説明はとても難しく感じるかもしれませんが、使い方は簡単です。 TasksSeeder.php の run メソッドを下記のように書き換えて、シードを実行してみてください。

    public function run()
    {
        factory(App\Task::class, 50)->create();
    }

php artisan db:seed

実行できたらデータベスを確認しましょう。 ダミーデータが50レコード追加されているのが確認できたと思います。

factory という関数が出てきましたね？これは factory ヘルパーと呼ばれるLaravelの機能になります。 factory ヘルパーを使うことで、ファクトリーに記載した内容を回数指定して実行することができました。 また、ファクトリーファイルの編集でも登場した App\Task というのは、モデルである Task.php までのnamespaceを指定しています。 これがないと、どのモデルに対するものなのかの特定ができなくなるので、間違えないように注意しましょう。
