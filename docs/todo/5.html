バリデーションその２

フォームバリデーション

より複雑なチェックを行う場合は、フォームリクエストを利用すると良いでしょう。
フォームリクエストとは、コントローラーで引数としてしてイた Request のことです。 フォームリクエストにはバリデーションを含めることができますが、カスタマイズすることによって 一層便利にバリデーションを実行する事ができます。

フォームリクエストの作成

フォームリクエストの作成をするには、ターミナルまたはコマンドプロンプトから下記のコマンドを実行します。

php artisan make:request StoreTaskPost

コマンドの実行に成功すると、app/Http/Request/StoreTaskPost.php が作成されます。 中を確認してみましょう。

<?php

namespace App\Http\Requests;

use Illuminate\Foundation\Http\FormRequest;

class StoreTaskPost extends FormRequest
{
    /**
     * Determine if the user is authorized to make this request.
     *
     * @return bool
     */
    public function authorize()
    {
        return false;
    }

    /**
     * Get the validation rules that apply to the request.
     *
     * @return array
     */
    public function rules()
    {
        return [
            //
        ];
    }
}

このファイル内の rulesメソッドにバリデーションを追加していくことになります。 とりあえず下記のようにしてルールを追加してみましょう。
また、まだ認証機能を実装していないので、authorizeメソッドもtrueに一時的に変更しておいてください。

    /**
     * Determine if the user is authorized to make this request.
     *
     * @return bool
     */
    public function authorize()
    {
        // 変更：認証のチェックをしない
        return true;
    }

    /**
     * Get the validation rules that apply to the request.
     *
     * @return array
     */
    public function rules()
    {
        // 変更ルールの追加
        return [
            'name' => 'required|max:100',
            'deadline' => 'required',
            'detail' => 'required',
        ];
    }

どのようにバリデーションルールを実行するのでしょうか？
下記の引数として Request の代わりに StoreTaskPost 使用することでやって来たフォームリクエストはコントローラメソッドが呼び出される前にバリデーションを行います。 つまり、コントローラにバリデーションロジックを書く必要がなくなるのです。

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;
use App\Http\Requests\StoreTaskPost; // <- これを追加
use App\Task;

class TodoController extends Controller
{

    ...（省略）...

    public function store (StoreTaskPost $request)
    {
        // コメントアウト
        // 追加：バリデーション
        // $request->validate([
        //     'name' => 'required|max:100',
        //     'detail' => 'required',
        // ]);

        $input = $request->all();
        unset($input['_token']);

修正できたら、早速ブラウザから動きを確認してみてください。 先程のバリデーションと同じ動きを確認できたと思います。

バリデーション属性のカスタマイズ

最後に、表示されるエラーメッセージのカスタマイズを行います。
現状、エラーとなったインプット項目は inputタグのものがそのまま表示されてしまいます。 これはユーザーにとっては優しくありません。そのため、ユーザーにわかりやすい名前に変更したいと思います。
StoreTaskPost.php に attributes メソッドを追加しましょう。

public function attributes()
{
    return [
        'name' => 'タスク名',
        'deadline' => 'タスク期限',
        'detail' => 'タスク詳細',
    ];
}

こうすることで、エラーメッセージ中の name を タスク名 などとして変更することができました。 実際にブラウザからエラーを発生させて確認してみてください。
