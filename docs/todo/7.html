アクセサとミューテータ

アクセサとミューテタ

https://readouble.com/laravel/5.8/ja/eloquent-mutators.html

[本文引用]:
アクセサとミューテタはモデルの取得や値を設定するときに、Eloquent属性のフォーマットを可能にします。

この説明ではよくわかりませんが、要するに取得したデータベースのカラムの情報を意図的にカスタマイズしたい時に アクセサやミューテタというのを有効に利用することができます。

アクセサ

例えば、Task.php の先頭に下記のようなメソッドを追記してください。

class Task extends Model
{
    protected $fillable = ['name', 'detail', 'deadline'];

    // 追加：アクセサ
    public function getDeadlineAttribute($value)
    {
        return date('Y-m-d',  strtotime($value));
    }

追加できたら、タスク編集画面をブラウザから確認してみてください。
どうでしょうか？初期値が表示されたのでは無いでしょうか？

アクセサとは、このように取得した情報を意図して操作することができます。

アクセサメソッドの命名規則

function get大文字から始まるカラム名Attribule($value)
{
    return 変更後の値;
}

ミューテタ

アクセサは取得した値をカスタマイズするものという説明を上記で書きましたが、ミューテタとはその逆で データベースに保存したい内容に変更を加えることができます。
例えば、Task.php の先頭に下記のようなメソッドを追記してください。

class Task extends Model
{
    protected $fillable = ['name', 'detail', 'deadline'];

    // 追加：ミューテタ
    public function setNameAttribute($value)
    {
        if ( ! preg_match('/^【Task】/', $value) ) {
            $this->attributes['name'] = '【Task】' . $value;
        } else {
            $this->attributes['name'] = $value;
        }
    }

    public function getDeadlineAttribute($value)
    {
        return date('Y-m-d',  strtotime($value));
    }

追加できたら、新規タスク登録画面にブラウザからアクセスして登録を確認してみてください。
登録したタスク名の先頭に「【Task】」という文字が追加されたのではないでしょうか？

ミューテタとは、このように登録する情報を意図して操作することができます。

ミューテタメソッドの命名規則

function set大文字から始まるカラム名Attribule($value)
{
    $this->attributes['カラム名'] = 変更後の値
}

アクセサと違う点は、必ずメソッド名に記載したカラムへの変更でなくても良いという点です。 ミューテタは、メソッド名に記載したカラムへの登録／更新処理が行われる場合に起動するメソッドとなります。
