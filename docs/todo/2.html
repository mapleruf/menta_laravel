タスク新規作成画面の作成

ルーティングとしては下記のURLを実行した場合の処理を実装していきます。

// タスク新規作成画面
Route::get('create', 'TodoController@create')->name('task.create');

ToDoControllerクラスのcreateメソッドにルーティングが設定されているのでTodoController.phpを編集していきます。
まずは、createメソッドを追加しましょう。

    public function create ()
    {

    }

ここにタスク追加画面の処理を書いていくのですが、今回は特に処理を行わないので そのままviewを表示してしまいましょう。

    public function create ()
    {
        return view('create');
    }

次にviewのテンプレートを作成していきます。
見た目を気にせずざっくりとフォームを作ってしまいましょう。

<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <title>タスク作成</title>
    <link rel="stylesheet" href="">
</head>
<body>
    <form action="{{route('task.store')}}" method="POST">
        @csrf
        タスク名：<input type="text" name="name" value="" placeholder=""><br>
        期日　　：<input type="date" name="deadline" value="" placeholder=""><br><br>
        詳細　　：<textarea name="detail"></textarea><br>
        <input type="submit" value="登録">
    </form>
</body>
</html>

ここでは、@csrf とだけ書かれた行があることに注目してください。
Laravelは、ユーザーの各セッションごとにCSRFトークンと呼ばれるものを自動的に生成しています。 そして、認証（ログイン）済みのユーザーがアプリケーションに対してリクエストを送信しているのかを確認するためにこのトークンを利用しています。

HTMLのフォームを定義する場合は、常にCSRF保護ミドルウェアがリクエストを検証できるように、 隠しCSRFトークンフィールドをそのフォームへ含める必要があります。
そのために @csrf を利用するのです。
