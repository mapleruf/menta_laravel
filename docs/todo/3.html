タスク登録処理の作成

タスク新規登録画面の作成が完了したら、早速タスクの登録処理を実装していきます。

// タスク登録処理
Route::post('/store', 'TodoController@store')->name('task.store');

上記の通り、ToDoControllerクラスのstoreメソッドにルーティングが設定されているのでTodoController.phpを編集していきます。
まずは、storeメソッドを追加しましょう。

    public function store (Request $request)
    {

    }

Request $request という引数が追加されていることに気づいたかもしれません。 この Request にはウェブフォームから送信された GET/POST のパラメータなどの情報が格納されています。

https://readouble.com/laravel/5.8/ja/requests.html

通常、特定のデータのみを取得したい場合は下記のようにしてGET/POSTされたデータを取得することができます。

$request->input('name');

これは、下記のようにinputタグに付けられた名前からGET/POSTされたデータを取得する方法です。

<form>
    <input type="text" name="name" value="" placeholder="">
    <input type="submit" name="" value="送信">
</form>

今回はPOSTされたデータ取得するために使用するため、１つづつ取得するのも良いですが一度にすべて取得してから 不要な情報を削除するという方法もあります。
今回はそちらを使用することにしましょう。

    public function store (Request $request)
    {
        $input = $request->all();
        unset($input['_token']);
    }

そしてデータベースにデータを書き込無処理を作っていきます。
本来であればモデルにデータベース操作に関するすべての業務を委託したいのですが、 簡単な作業であればコントローラーで行っても問題はありません。
適材適所ではないですが、読みやすいコード（Readable Code）を意識してコーディングするようにしましょう。

    public function store (Request $request)
    {
        $input = $request->all();
        unset($input['_token']);

        // 追加：データベースへの登録
        $tasks = new Task;
        $tasks->fill($input);
        $tasks->save();

        return redirect(route('task.index'));
    }

早速、php artisan serve して画面から登録を実行してみましょう。 この実行は必ず下記のようなエラーになるはずです。

Illuminate \ Database \ Eloquent \ MassAssignmentException
Add [name] to fillable property to allow mass assignment on [App\Task].

これは、fill というメソッドを使っている場合に許可されていないカラムへの書き込みが実行されたことを意味しています。 これを解決するためにTasks.phpに行を追記してください。

<?php

namespace App;

use Illuminate\Database\Eloquent\Model;

class Task extends Model
{
    // 追加：fill可能なカラム
    protected $fillable = ['name', 'detail', 'deadline'];

追加が終わったらもう一度、登録を実行してみましょう。
