バリデーションその１

バリデーション

https://readouble.com/laravel/5.8/ja/validation.html

この章では、バリデーションについて学びます。 バリデーションとは入力チェックのようなもので、入力されたデータが意図した形式や 不正なもので無いことをチェックする仕組みです。

バリデーションのためのロジックは Request クラスに含まれていてコントローラー毎に その内容を設定することができます。

    public function store (Request $request)
    {
        // 追加：バリデーション
        $request->validate([
            'name' => 'required|max:100',
            'deadline' => 'required',
            'detail' => 'required',
        ]);

        $input = $request->all();
        unset($input['_token']);

このままブラウザで実行しても問題はありませんが、エラーメッセージを表示することができないため 何のエラーかを特定できません。エラーメッセージを表示するには、$errors ディレクティブを 利用します。
create.blade.php を下記のように変更してみてください。

<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <title>タスク作成</title>
    <link rel="stylesheet" href="">
</head>
<body>
{{-- 追加：エラー表示 --}}
@if ( $errors->any() )
    <p>
    @foreach ($errors->all() as $err)
    {{ $err }}<br>
    @endforeach
    </p>
@endif
    <form action="{{route('task.store')}}" method="POST">
        @csrf
        タスク名：<input type="text" name="name" value="" placeholder=""><br>
        期日　　：<input type="date" name="deadline" value="" placeholder=""><br><br>
        詳細　　：<textarea name="detail"></textarea><br>
        <input type="submit" value="登録">
    </form>
</body>
</html>

修正が終わったら、「タスク名」と「詳細」を入力せず登録を実行してみましょう。
英語でエラーメッセージは表示されたと思います。しかし、日本語ではないので読みづらいですね。

バリデーションの日本語化

これを解消するために、Laravelの設定を変更します。 config/app.phpをエディタで開いて下記の行を見つけてください。

    /*
    |--------------------------------------------------------------------------
    | Application Locale Configuration
    |--------------------------------------------------------------------------
    |
    | The application locale determines the default locale that will be used
    | by the translation service provider. You are free to set this value
    | to any of the locales which will be supported by the application.
    |
    */

    'locale' => 'en',

設定には en と書いてあるためLaravelの設定は英語です。これを ja に変更してみましょう。

    /*
    |--------------------------------------------------------------------------
    | Application Locale Configuration
    |--------------------------------------------------------------------------
    |
    | The application locale determines the default locale that will be used
    | by the translation service provider. You are free to set this value
    | to any of the locales which will be supported by the application.
    |
    */

    'locale' => 'ja',

次に、resources/lang/en/ というフォルダをそのままコピーして resources/lang/ja に変更します。
イメージとしては下記のようにenフォルダとjaフォルダが存在している状態にします。

resources
 └── lang
        ├── en
        └── ja

これでバリデーションの際には、jaフォルダに配置されているファイルを元にメッセージを表示するように 変更する事ができました。 しかし、コピーしただけなのでこのままでは英語で表示されてしまいますね。
そのため、validation.phpの内容を手動で変更していく必要があります。

... (省略) ...
    'boolean' => ':attribute は、TRUEかFALSEで指定する必要があります.',
    'confirmed' => ':attribute の確認項目が一致しません.',
    'date' => ':attribute が不正な日付です.',
... (省略) ...

ちょっと面倒くさいですよね（汗
このファイルについては既に翻訳したものを公開していたりもしますので、それを活用することで 簡単に変更することもできます。

単体バリデーションメッセージの表示

先程のサンプルの例ではすべてのエラーを一度に表示していますが、単体で表示させることもできます。
これはフォームのパーツごとにエラーを表示させたい場合などに有効です。

<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <title>タスク作成</title>
    <link rel="stylesheet" href="">
</head>
<body>
    <form action="{{route('task.store')}}" method="POST">
        @csrf
        タスク名：<input type="text" name="name" value="" placeholder=""><br>
        @if ($errors->has('name') )
        <span>{{ $errors->first('name') }}</span><br>
        @endif
        期日　　：<input type="date" name="deadline" value="" placeholder=""><br>
        @if ($errors->has('deadline') )
        <span>{{ $errors->first('deadline') }}</span><br>
        @endif
        詳細　　：<textarea name="detail"></textarea><br>
        @if ($errors->has('detail') )
        <span>{{ $errors->first('detail') }}</span><br>
        @endif
        <input type="submit" value="登録">
    </form>
</body>
</html>

$errors->has() は、渡された引数のインプット項目に関するエラーが有るかどうかを判定します。 損刺しした場合、$errors->first() を使って特定のインプット項目に関するエラーメッセージを 取得する事ができます。
